# 변화관리 (Change Readiness — 7 Success Factors)

"이 변화 프로젝트, 성공할 수 있을까? 어디가 취약한가?"를 진단하는 도구.

---

## FRAMEWORK

Ulrich의 변화 성공 7요인 프로파일링 도구 (Ch6, Table 6-3).

Change Agent 역할의 핵심 도구 — 특정 변화 이니셔티브의 성공 가능성을 7가지 요인으로 사전 진단한다.

### 7가지 변화 성공 요인

| # | 성공 요인 | 핵심 질문 |
|---|----------|----------|
| 1 | **Leading Change** | 변화를 책임질 스폰서가 있는가? |
| 2 | **Creating Shared Need** | 직원들이 변화의 필요성을 느끼는가? |
| 3 | **Shaping a Vision** | 변화 후 미래 비전이 명확한가? |
| 4 | **Mobilizing Commitment** | 핵심 이해관계자가 헌신하고 있는가? |
| 5 | **Changing Systems** | HR 시스템·구조가 변화를 지원하는가? |
| 6 | **Monitoring Progress** | 진행 상황 측정 벤치마크가 있는가? |
| 7 | **Making Change Last** | 변화가 지속적 실행으로 이어지는가? |

### 3가지 변화 유형

| 유형 | 초점 | 예시 |
|------|------|------|
| **Initiative** | 새 프로그램·프로젝트 실행 | 새 조직구조, 고객서비스 개선 |
| **Process** | 업무 수행 방식 변화 | GE Workout, 프로세스 재설계 |
| **Cultural** | 핵심 가치·정체성 변화 | GE 전사 문화 변혁, Sears 전환 |

> Initiative → Process → Cultural 순으로 누적·순환

### 변화 실패 8가지 원인 (Table 6-2)

1. 긴급하지 않음 2. 단기적 관점 3. 정치적 동기 4. 경직된 설계
5. 드문 측정 6. 획일적 접근 7. 헌신 동원 실패 8. 변화 지속 실패

### 3가지 문화 변화 접근법

| 접근법 | 방향 | 예시 |
|--------|------|------|
| **Top-Down** | 경영진 → 하향 전파 | Xerox Leadership Through Quality |
| **Side-to-Side** | 횡적 프로세스 재설계 | Northern Telecom 행정 간소화 |
| **Bottom-Up** | 직원이 직접 혁신 | GE Workout Town Meeting |

## EXECUTE

### 입력 요청

사용자에게 아래 정보를 요청한다:

1. **변화 프로젝트명** — 무엇을 변화시키려 하는가?
2. **배경** — 왜 이 변화가 필요한가?
3. **현재 상태** — 어디까지 진행되었는가?
4. **관련자** — 스폰서, 지지자, 저항자는 누구인가?
5. **변화 유형** — Initiative / Process / Cultural 중 어디에 해당?

### 분석 내용

1. **7요인 프로파일링** — 각 요인별 현재 수준(0~100) 추정 + 근거
2. **취약 요인 식별** — 실패 위험이 가장 높은 요인
3. **실패 원인 매칭** — 8가지 실패 원인 중 해당하는 것
4. **다중 관점 분석** — 경영진/직원/HR 최소 2개 관점에서 동일 7요인 진단
5. **접근법 추천** — Top-Down / Side-to-Side / Bottom-Up 중 적합한 것
6. **액션플랜** — 취약 요인 보강을 위한 구체적 행동

### 다중 관점 분석 구조

```
관점 1: [경영진/전략 컨설턴트] → 7요인 진단
관점 2: [현장 직원/조직심리학자] → 7요인 진단
관점 3: [HR/변화관리 전문가] → 7요인 진단
     ↓
합의점: 모든 관점에서 동의하는 부분
충돌점: 관점 간 평가가 다른 부분 (★ 핵심 인사이트)
종합 권고: 충돌을 해결하는 방향의 액션
```

## CC-CONCEPT

**연결 개념:** Subagent + Team Agent

> "Change Agent는 혼자 변화를 이끌지 않아요. 리더, 직원, 외부 컨설턴트를 함께 움직여야 하죠. Claude Code에서도 하나의 AI에게만 물어보는 게 아니라, 여러 전문가 관점의 AI를 동시에 투입할 수 있어요."

**Subagent란:**
- CC 안에서 별도의 AI 전문가를 생성해서 특정 관점으로 분석을 맡기는 것
- 각 Subagent는 독립적으로 작업하고 결과를 보고

**Team Agent란:**
- 여러 Subagent를 하나의 팀으로 묶어서 총괄하는 리더 AI
- 개별 분석을 종합하고, 관점 간 충돌을 정리

**시각적 설명:**

```
[Team Agent: 변화관리 총괄]
    ├── Subagent 1: 전략 컨설턴트 → 7요인 경영진 관점
    ├── Subagent 2: 조직심리학자 → 7요인 직원 관점
    └── Subagent 3: HR 전문가 → 7요인 HR 관점
         ↓
    종합 리포트: 합의점 + 충돌점 + 권고안
```

**실습 안내:**

> CC에 이렇게 요청해보세요: "이 변화 프로젝트를 3가지 관점에서 분석해줘. 1) 경영진 관점 2) 직원 관점 3) HR 관점. 각각 7요인으로 진단하고 의견이 다른 부분을 알려줘."
