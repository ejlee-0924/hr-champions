# HR 역할 진단 (40문항 Role-Assessment)

"나는 어떤 HR 역할이 강하고, 어디를 개발해야 하는가?"를 진단하는 도구.

---

## FRAMEWORK

Ulrich의 40문항 HR Role-Assessment Survey (Ch2 부록).

**구조:** 10개 카테고리 x 4문항 = 40문항, 1(낮음)~5(높음) 5점 척도

**채점 키:**

| 역할 | 해당 문항 |
|------|----------|
| Strategic Partner | Q1, 5, 9, 13, 17, 21, 25, 29, 33, 37 |
| Admin Expert | Q2, 6, 10, 14, 18, 22, 26, 30, 34, 38 |
| Employee Champion | Q3, 7, 11, 15, 19, 23, 27, 31, 35, 39 |
| Change Agent | Q4, 8, 12, 16, 20, 24, 28, 32, 36, 40 |

**해석 기준:**
- 역할별 만점: 50점 (10문항 x 5점)
- 총점 160+ → HR 기능 품질 높음
- 총점 90 미만 → HR 기능 품질 낮음

**핵심 활용:** HR 전문가 + 라인 매니저가 별도 평가 → 점수 비교 → 불일치 영역이 대화 출발점

---

## EXECUTE

### 입력 방식 A: 역할별 합산 점수 직접 입력

사용자에게 아래를 요청한다:

```
4역할별 점수를 알려주세요 (각 10~50점 범위):
- Strategic Partner: ?
- Administrative Expert: ?
- Employee Champion: ?
- Change Agent: ?
```

### 입력 방식 B: 40문항 개별 응답

사용자가 40문항 전체를 응답하면, 채점 키에 따라 역할별로 합산한다.

### 분석 내용

1. **4역할 점수 비교** — 강점 역할 / 개발 필요 역할 식별
2. **균형 분석** — 전략-운영 축, 프로세스-사람 축 편향 진단
3. **역설 분석** — Strategic Partner vs Employee Champion, Change Agent vs Admin Expert 간 긴장 관계
4. **개발 우선순위** — 가장 낮은 역할에 대한 구체적 강화 방향 제안
5. **다음 도구 추천** — 가장 약한 역할에 해당하는 실무 도구 안내

### 리포트 형식

```markdown
# 나의 HR Champions 역할 진단

**날짜:** [YYYY-MM-DD]

## 점수 요약

| 역할 | 점수 | 수준 |
|------|------|------|
| Strategic Partner | /50 | |
| Admin Expert | /50 | |
| Employee Champion | /50 | |
| Change Agent | /50 | |
| **총점** | **/200** | |

## 강점 역할
[가장 높은 점수 역할 분석]

## 개발 필요 역할
[가장 낮은 점수 역할 분석 + 강화 방향]

## 균형 분석
- 전략 vs 운영: [편향 진단]
- 프로세스 vs 사람: [편향 진단]

## 추천 다음 단계
[약한 역할에 해당하는 도구 안내]
```

## CC-CONCEPT

**연결 개념:** 프롬프팅 기초

> "이 진단에서 여러분이 한 게 프롬프팅의 핵심이에요. 맥락(내 점수) + 요청(분석해줘) + 형식(강점/개발 구분)을 줬더니 CC가 의미 있는 결과를 냈죠."

프롬프팅 3요소:
1. **맥락(Context)** — 내가 누구이고, 어떤 상황인지
2. **요청(Request)** — 무엇을 해달라는 것인지
3. **형식(Format)** — 어떤 형태로 결과를 원하는지
