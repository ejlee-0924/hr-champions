# 프로세스 개선 (HR Process Reengineering)

"이 HR 프로세스를 어떻게 효율화하고 자동화할 수 있을까?"를 분석하는 도구.

---

## FRAMEWORK

Ulrich의 HR 리엔지니어링 6단계 로드맵 (Ch4, Table 4-2).

Administrative Expert 역할의 핵심 도구 — HR 프로세스를 진단하고, 비효율을 제거하고, 자동화 가능한 영역을 식별한다.

### 6단계 로드맵

| 단계 | 활동 | 결과물 |
|------|------|--------|
| 1. 목표 정의 | 프로세스 정의, 고객 인터뷰 | 목적·고객·이해관계자 정의 |
| 2. 현황 분석 | 업무 흐름 문서화, 비효율 파악 | 문제 진단 |
| 3. 벤치마크 | 외부 비교, 목표 설정 | 갭 분석 |
| 4. "Should Be" 설계 | 대안 탐색, 비용편익 분석 | 새 프로세스 설계 |
| 5. 구현/파일럿 | 점진적 구현, 저항 관리 | 사용자 수용도 |
| 6. 임팩트 측정 | 비용·품질·속도 측정 | 모니터링·영향 측정 |

### 벤치마크 참고 (Table 4-3)

| 기업 | 성과 |
|------|------|
| 3M | HR 비용 30% 절감 (5년) |
| Calvin Pacific | HR 인원 55명→21명 (2년) |
| HP | HR 비율 1:50→1:75 |
| Baxter | HR 비율 1:50→1:190 (90% 디지털화) |

### 6가지 HR 전달 메커니즘 (Figure 4-2)

| 메커니즘 | 핵심 | 적합 상황 |
|---------|------|----------|
| **Corporate Staff** | 기업 전체 정책·표준화 | 일관성 필요 시 |
| **BU Staff** | 사업부 전용 HR | 사업부 자율성 필요 시 |
| **Broker of Services** | 외주 계약 서비스 | 전문성 부족 시 |
| **Service Center** | 백오피스 표준화 운영 | 규모 효율화 시 |
| **Center of Expertise** | 세계 수준 전문성 | 깊은 전문성 필요 시 |
| **Integrated Solutions** | 통합 컨설팅 제공 | 복합 문제 해결 시 |

## EXECUTE

### 입력 요청

사용자에게 아래 정보를 요청한다:

1. **프로세스명** — 어떤 HR 프로세스를 분석할 것인지
2. **현재 흐름** — 단계별 현황 (누가, 무엇을, 어떻게, 얼마나 걸리는지)
3. **고객** — 이 프로세스의 최종 사용자/수혜자는 누구인지
4. **문제점** — 현재 겪고 있는 비효율/불만

### 분석 내용

1. **6단계 순차 분석** — 각 단계별 현황 진단
2. **비효율 식별** — 수작업, 중복, 병목, 대기 시간
3. **자동화 가능 영역** — AI/자동화로 대체 가능한 단계 식별
4. **Should-Be 설계** — 개선된 프로세스 플로우
5. **예상 ROI** — 시간 절감, 비용 절감, 품질 향상 추정
6. **전달 메커니즘 추천** — 6가지 중 적합한 모델

### 리포트 형식

```markdown
# HR 프로세스 리엔지니어링 분석

**대상 프로세스:** [프로세스명]
**날짜:** [YYYY-MM-DD]
**프레임워크:** Ulrich HR 리엔지니어링 6단계 (Ch4, Table 4-2)

## 1. 목표 정의
- 프로세스 목적: ...
- 고객: ...
- 이해관계자: ...

## 2. 현황 분석 (As-Is)
[단계별 현재 흐름 + 소요 시간/인력]

## 3. 벤치마크
[업계 평균 대비 + 개선 목표]

## 4. Should-Be 설계
[개선된 프로세스 플로우]

### 자동화 가능 영역
| 단계 | 현재 | 개선 | 자동화 방식 |
|------|------|------|-----------|

## 5. 구현 계획
[파일럿 범위 + 일정 + 저항 관리]

## 6. 예상 임팩트
- 시간 절감: ...
- 비용 절감: ...
- 품질 향상: ...
```

## CC-CONCEPT

**연결 개념:** Skill (재사용 가능한 자동화 레시피)

> "Admin Expert의 핵심은 '매번 수작업으로 하던 걸 시스템으로 바꾸는 것'이에요. Claude Code에서 이걸 하는 방법이 Skill입니다. 한 번 만들어두면 `/스킬이름`만 치면 자동 실행돼요."

**Skill이란:**
- SKILL.md 파일 하나로 CC의 행동 패턴을 정의
- "이런 요청이 오면 → 이런 단계로 분석해라"는 레시피
- 한 번 만들면 반복 사용 가능, 팀에 공유 가능

**실습 안내:**

> 방금 한 리엔지니어링 분석을 Skill로 만들어볼까요? 이렇게 하면 다른 HR 프로세스에도 `/리엔지니어링`만 치면 같은 분석을 할 수 있어요.

```markdown
# SKILL.md 예시

---
name: hr-reengineering
description: HR 프로세스를 입력받아 리엔지니어링 6단계로 분석
---

이 스킬은 HR 프로세스를 입력받아 Ulrich의 리엔지니어링
6단계로 분석하고 개선안을 제시합니다.

## 실행 방법
1. 대상 프로세스 현황을 입력받는다
2. 6단계로 분석한다
3. 자동화 가능 영역 + 예상 ROI 포함 리포트 생성
```
