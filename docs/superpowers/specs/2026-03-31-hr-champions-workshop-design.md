# HR Champions x Claude Code 워크샵 설계

**날짜:** 2026-03-31
**상태:** 설계 완료, 사용자 검토 대기

---

## 1. 개요

### 컨셉
**"HR Champions x AI: Ulrich 4역할을 Claude Code로 실행하기"**

Dave Ulrich의 HR Champions(1997) 4-Role 모델을 학습하면서, 각 역할에 대응하는 Claude Code 핵심 개념을 동시에 익히는 워크샵. HR 프레임워크가 목적이고, Claude Code는 그 실현 도구.

### 핵심 차별점
- 기존 `cc-tutor-hr`: CC 도구 사용법을 HR 시나리오로 감싸는 구조
- **이 워크샵**: HR Champions 프레임워크를 실행하는 수단으로 CC를 배우는 구조

### 연결 OKR
- O4 (Thought Leadership): 외부 HR 대상 오픈 강의로 업계 영향력 확보
- O1 (AX 패러다임): AI 도구 활용 역량 외부 확산

## 4. 실습 설계 원칙

**가상 케이스 시연 → 본인 회사 적용** (매 역할 동일 패턴)

| 단계 | 시간 | 목적 |
|------|------|------|
| 가상 케이스 시연 | 15분 | 퍼실리테이터가 A사 데이터로 시연 → "이렇게 하면 되는구나" 레시피 제공 |
| 본인 회사 적용 | 20분 | 참여자가 자기 회사 맥락으로 동일 실습 → 실전 산출물 생성 |
| 공유 | 5분 | 2~3명 자발적 공유 → 상호 학습 |

**가상 기업 설정:**
- A사: 중견 제조업, 매출 3,000억, 직원 800명
- 제조+영업+R&D 3개 본부, 동남아 법인 신설 중
- 이직률 15%, 부서 간 사일로, 리더십 파이프라인 부재

## 5. 워크샵 타임라인 (4시간)

### 블록 1: 오프닝 (0:00-0:30)

**목표:** HR Champions 프레임워크 이해 + CC 첫 체험

| 시간 | 내용 | 방식 |
|------|------|------|
| 0:00-0:05 | 아이스브레이킹 | "HR 담당자로서 가장 많은 시간을 쓰는 업무는?" (손들기/채팅) |
| 0:05-0:15 | HR Champions 핵심 소개 | Deliverables vs Doables → 4역할 2x2 매트릭스 → "활동이 아니라 산출물로 측정" |
| 0:15-0:20 | 왜 지금 이 책인가 | 1997년 프레임워크가 AI 시대에 더 유효한 이유 — "Admin Expert의 리엔지니어링 대상이 이제 AI" |
| 0:20-0:30 | CC 소개 + 첫 체험 | CC란 무엇인가 + 첫 대화: "안녕, 나는 OO회사 HR 담당자야" |

**4역할 매트릭스 (슬라이드용):**

```
              전략 (미래)
                │
  Strategic     │     Change
  Partner       │     Agent
  (전략 실행)    │     (변혁 역량)
                │
 프로세스 ──────┼────── 사람
                │
  Admin         │     Employee
  Expert        │     Champion
  (효율 인프라)  │     (직원 몰입)
                │
              운영 (일상)
```

**퍼실리테이터 멘트:**
> "이 4가지를 다 잘하는 HR은 없어요. 중요한 건 '나는 어디가 강하고 어디가 약한지' 아는 겁니다. 그래서 지금부터 진단해볼 거예요."

---

### 블록 2: 40문항 진단 (0:30-0:50)

**목표:** 자기 진단 + CC 프롬프팅 첫 실전 체험
**HR 도구:** Ulrich 40문항 HR Role-Assessment Survey
**CC 개념:** 프롬프팅 기초 (맥락 + 요청 + 형식)

**실습 흐름:**

1. 퍼실리테이터가 40문항을 화면에 띄움 (또는 인쇄물/구글폼 배포)
2. 참여자가 1~5점 체크 (10분)
3. 역할별 합산 후 CC에 첫 실전 요청:

> **참여자 입력 예시:**
> "나의 HR Role Assessment 결과야. Strategic Partner: 38, Admin Expert: 27, Employee Champion: 42, Change Agent: 31. 이걸 분석해서 나의 강점 역할과 개발 필요 역할을 알려줘."

**CC 산출물:**
- 4역할 점수 비교 분석
- 강점 역할 / 개발 필요 역할 식별
- "오늘 워크샵에서 특히 집중할 역할" 추천

**퍼실리테이터 멘트:**
> "방금 여러분이 한 게 프롬프팅이에요. 맥락(내 점수) + 요청(분석해줘) + 형식(강점/개발 구분)을 줬더니 CC가 의미 있는 결과를 냈죠."

---

### 블록 3: Strategic Partner + CLAUDE.md (0:50-1:30)

**목표:** 조직 전략 맥락을 CC에 심어주는 법 학습
**HR 도구:** 조직 아키텍처 6요소 (Shared Mindset, Competence, Consequence, Governance, Work Process, Leadership)
**CC 개념:** CLAUDE.md — 프로젝트의 전략적 맥락을 담는 파일

**비유 도입:**
> "Strategic Partner는 비즈니스 전략을 깊이 이해해야 합니다. CC도 마찬가지예요. 여러분 회사를 모르는 CC에게 전략적 맥락을 알려주는 파일이 CLAUDE.md입니다."

**가상 케이스 시연 (15분):**

퍼실리테이터가 A사 CLAUDE.md를 보여주며 작성:

```markdown
# CLAUDE.md — A사 (가상 중견 제조업)

## 회사 개요
- 매출 3,000억, 직원 800명, 제조+영업+R&D 3개 본부
- 최근 동남아 법인 설립, 글로벌 확장 중

## 현재 이슈
- 이직률 15% (업계 평균 10%)
- 신규 사업부 신설 후 부서 간 사일로 심화
- 리더십 파이프라인 부재

## HR 전략 방향
- 핵심 인재 리텐션 강화
- 크로스펑셔널 협업 문화 구축
```

CC에 요청: "이 조직을 Ulrich의 조직 아키텍처 6요소로 진단해줘. 각 요소별 1~10점 추정 + 근거 + 우선 개선 영역 3개를 알려줘."

→ CC가 6요소 진단 리포트 생성

**본인 회사 적용 (20분):**
1. 본인 회사 CLAUDE.md 작성 (회사 개요 + 현재 이슈 + HR 방향)
2. CC에 조직 아키텍처 6요소 진단 요청
3. 결과 확인

> "방금 만든 CLAUDE.md가 앞으로 모든 실습의 기반이 됩니다. Strategic Partner가 비즈니스 맥락 위에서 일하듯, CC도 이 파일 위에서 일해요."

**공유 (5분):** "제 회사에서 가장 취약한 요소는 OO이었어요"

### 블록 4: Admin Expert + Skill (1:40-2:20)

**목표:** 반복되는 HR 프로세스를 자동화 레시피(Skill)로 만드는 법 학습
**HR 도구:** HR 리엔지니어링 6단계 로드맵 (목표정의 → 현황분석 → 벤치마크 → Should-Be 설계 → 구현 → 임팩트 측정)
**CC 개념:** Skill — 재사용 가능한 자동화 레시피

**비유 도입:**
> "Admin Expert의 핵심은 '매번 수작업으로 하던 걸 시스템으로 바꾸는 것'이에요. CC에서 이걸 하는 방법이 Skill입니다. 한 번 만들어두면 `/내스킬이름`만 치면 자동 실행돼요."

**가상 케이스 시연 (15분):**

A사의 채용 프로세스 현황:
- 이력서 접수: 메일로 수신 → 수작업 분류 (3일)
- 1차 스크리닝: 담당자가 이력서 하나씩 읽고 판단 (5일)
- 면접 조율: 전화+메일 왕복 평균 5회 (3일)
- 합격 통보: 개별 메일 수작업
- 총 소요: 평균 16일, HR 담당자 주당 15시간 투입

CC에 요청: "이 채용 프로세스를 Ulrich의 리엔지니어링 6단계로 분석해줘. 현황 진단 → 벤치마크 → Should-Be 설계 → 자동화 가능 영역 → 예상 비용 절감을 포함해줘."

→ CC가 리엔지니어링 보고서 생성

Skill 구조 시연:

```markdown
# SKILL.md — HR 프로세스 리엔지니어링 분석기

이 스킬은 HR 프로세스를 입력받아 Ulrich의 리엔지니어링
6단계로 분석하고 개선안을 제시합니다.

## 실행 방법
1. 대상 프로세스 현황을 입력받는다
2. 6단계(목표정의→현황분석→벤치마크→설계→구현→측정)로 분석
3. 자동화 가능 영역 + 예상 ROI 포함 리포트 생성
```

> "이 파일 하나 만들어두면, 앞으로 어떤 HR 프로세스든 `/리엔지니어링` 한 번으로 분석할 수 있어요."

**본인 회사 적용 (20분):**
1. 본인 회사에서 가장 비효율적인 HR 프로세스 하나를 CC에 설명
2. CC가 리엔지니어링 6단계 분석 실행
3. (시간 여유 시) 그 분석 프롬프트를 Skill로 저장 체험

**공유 (5분):** "어떤 프로세스를 분석했고, CC가 뭘 제안했나요?"

### 블록 5: Employee Champion + MCP/Plugin (2:20-3:00)

**목표:** 외부 시스템 데이터를 CC로 가져오는 MCP 개념 이해
**HR 도구:** 10C 자원 진단 (Control, Commitment, Challenging Work, Collaboration, Culture, Compensation, Communication, Concern for Due Process, Computers, Competence)
**CC 개념:** MCP — 외부 시스템의 데이터를 CC로 연결하는 통로

**비유 도입:**
> "Employee Champion은 직원의 목소리를 '듣는' 역할이에요. 그런데 목소리가 Slack에도 있고, 설문에도 있고, 이메일에도 있죠. MCP는 이 흩어진 채널들을 CC에 연결해서 한 번에 들을 수 있게 해줍니다."

**가상 케이스 시연 (15분):**

A사 직원 인터뷰 요약 3건 + 만족도 설문 결과:

```
[인터뷰 1 - 영업팀 김대리]
"야근이 많은데 결정권이 없어요. 위에서 시키는 대로만..."

[인터뷰 2 - R&D팀 박주임]
"새 기술 배울 기회가 없어요. 매일 같은 일만 반복..."

[인터뷰 3 - 경영지원팀 이과장]
"팀 간 협업이 안 돼요. 각자 사일로에서 일하는 느낌..."

[설문 결과 요약]
전체 만족도 3.2/5, 성장 기회 2.8/5, 소통 2.5/5
```

CC에 요청: "이 직원 데이터를 Ulrich의 10C 프레임워크로 분석해줘. 각 C별로 시그널을 매핑하고, Demands > Resources인 영역을 식별해서 개선 우선순위를 알려줘."

→ CC가 10C 매핑 분석 + 갭 진단 리포트 생성

MCP 개념 시각적 설명:

```
[Slack] ──┐
[설문] ───┤── MCP ──→ CC ──→ 10C 진단 리포트
[메일] ───┘
```

> "Employee Champion이 여러 채널에서 직원 목소리를 수집하듯, MCP는 CC의 귀를 여러 시스템에 연결해주는 겁니다."

**본인 회사 적용 (20분):**
1. 본인 팀/회사의 직원 이슈를 텍스트로 CC에 입력 (인터뷰 기억, 설문 결과, 퇴사자 면담 등)
2. CC가 10C 프레임워크로 분석
3. "우리 회사에서 MCP로 연결하면 좋을 시스템은?" CC에 질문

**공유 (5분):** "10C 중 가장 취약했던 C는 뭐였나요?"

### 블록 6: Change Agent + Subagent/Team (3:10-3:50)

**목표:** 다중 전문가 AI를 동시에 투입하는 Subagent/Team 개념 이해
**HR 도구:** 변화 성공 7요인 체크리스트
**CC 개념:** Subagent + Team Agent — 여러 전문가 AI의 병렬 분석

**비유 도입:**
> "Change Agent는 혼자 변화를 이끌지 않아요. 리더, 직원, 외부 컨설턴트를 함께 움직여야 하죠. CC에서도 하나의 AI에게만 물어보는 게 아니라, 여러 전문가 관점의 AI를 동시에 투입할 수 있어요. 이게 Subagent이고, 팀으로 묶으면 Team Agent입니다."

**가상 케이스 시연 (15분):**

A사의 변화 프로젝트 — "전사 AI 도입 추진":
- CEO가 선언했지만 현장 저항 심함
- IT팀만 앞서가고 나머지 부서는 관망
- 중간 관리자가 "내 일자리 위협" 인식
- 예산은 확보, 하지만 성공 지표 불명확

CC에 요청: "이 변화 프로젝트를 3가지 관점에서 동시에 분석해줘. 1) 경영진 관점의 전략 컨설턴트 2) 현장 직원 관점의 조직심리학자 3) HR 관점의 변화관리 전문가. 각각 Ulrich의 변화 성공 7요인으로 진단하고, 의견이 다른 부분을 하이라이트해줘."

→ CC가 3개 관점의 병렬 분석 + 의견 충돌 지점 식별

Subagent/Team 개념 시각화:

```
[Team Agent: 변화관리 총괄]
    ├── Subagent 1: 전략 컨설턴트 → 7요인 경영진 관점
    ├── Subagent 2: 조직심리학자 → 7요인 직원 관점
    └── Subagent 3: HR 전문가 → 7요인 HR 관점
         ↓
    종합 리포트: 합의점 + 충돌점 + 권고안
```

**본인 회사 적용 (20분):**
1. 본인 회사에서 진행 중이거나 계획 중인 변화 프로젝트를 CC에 설명
2. CC에게 다중 관점 분석 요청 (최소 2개 관점)
